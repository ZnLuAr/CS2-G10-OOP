# dev-materials-for-report/

本目录用于存放**面向最终报告**的开发素材：决策记录、开发日志、测试笔记、接口规范等。

> 这些文件的内容会在期末报告中被大量引用——边开发边写，比临到 deadline 才回忆要轻松得多。

## 文件总览

| 文件 | 用途 | 对应报告章节 |
|------|------|--------------|
| [`design-decisions.md`](./design-decisions.md) | 设计决策记录："为什么这么做" | 技术设计 / 系统设计 |
| [`development-log.md`](./development-log.md) | 时间线式开发日志：什么时候改了什么 | 开发与测试 |
| [`testing-notes.md`](./testing-notes.md) | 测试与调试笔记：发现的 bug 与修复过程 | 开发与测试 |

> 项目级的设计文档（`data-design.md` / `error-and-log-design.md` / `功能列表.csv` 等）在 [`docs/` 根目录](../)，不在本目录下。

## 与外层 docs/ 的分工

- **`docs/` 根目录**：面向"项目本身"的稳定文档（数据设计、功能列表、项目说明、反思等）
- **本子目录**：面向"开发过程 / 报告素材"的文档（迭代记录、决策推演、bug 排查）

## 写作建议

- **当场记**：决策、踩坑、反思应该开发过程中**当场**写进对应文档，不要拖到最后回忆
- **统一格式**：每条记录建议带日期 `[YYYY-MM-DD]`，便于追溯
- **直接面向报告**：写文档时就当作是在写报告，最终可以直接复用内容
