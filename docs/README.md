## 使用建议

- **边开发边记**：开发过程中产生的设计决策、踩坑、反思应该**当场**写进对应文档，不要拖到最后回忆 ~~（实在不行可以让 AI 帮忙）~~
- **统一风格**：每条记录建议带日期 `[YYYY-MM-DD]`，便于追溯
- **直接面向报告**：写文档时就当作是在写报告，最终可以直接复用内容
