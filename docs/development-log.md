# 开发日志

> 时间线式记录关键变更：什么时候改了什么、为什么改。
> 用于报告的"开发与测试"章节，证明系统是迭代演进的。

---

## 模板

### [YYYY-MM-DD] 简短标题

- **变更内容**：做了什么
- **原因**：为什么要做
- **遗留问题**：（可选）有没有引入新问题或待办

---

## 日志

### [YYYY-MM-DD] 示例：初始化项目骨架

- **变更内容**：创建 `src/` 目录结构、空的模块入口、`main.py`
- **原因**：让后续开发有清晰的位置可以放代码
- **遗留问题**：暂未确定 `services/` 和 `models/` 的边界，待第一个 feature 实装时再细化
