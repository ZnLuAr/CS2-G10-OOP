JC1503 面向对象编程
小组项目——个人数据系统
小组人数：4–6人
编程语言：Python
课程协调人：李睿哲、沙赫扎德·穆姆塔兹、于振华
1. 项目说明
本小组项目的目标是设计并提交你们团队当前最优版本的Python个人软件系统，并展示你能够以JC1503课程所授核心思想进行开发。本项目不只要求程序可运行，更要体现你能以清晰结构、明确的类职责、合理的数据结构选择来设计软件系统，确保系统具备实际可扩展性与可维护性。
最终提交物需是选定领域内完整、连贯的应用程序（例如记录管理系统、追踪系统、预订系统或其他真实场景）。系统应能完成多项任务，支持一组有意义的操作（新增、修改、删除、查询、安全异常处理），整体表现为一个完整应用，而非零散脚本的集合。
请尽可能结合自身开发过程论证设计决策。例如：解释为何用特定类建模问题、为何选择特定关系（继承 vs 组合）、为何特定数据结构适用于对应操作。同时需用测试与调试结果支撑决策：测试内容、出现的问题、定位与修复方式。目的是证明系统并非一蹴而就，而是经过迭代开发、优化与完善。
核心要求：系统需以真实方式创建运行环境与数据。首次运行时初始化数据集并保存到文件；此后每次运行加载并更新已有数据，而非重新生成。这让应用更贴近真实系统：具备状态、可迭代、可重复使用。
主要提交物为可运行的软件系统。书面报告与演示视频用于说明并论证系统的设计与开发过程。
本项目为小组提交。由一名组员代表全组提交。确保报告清晰列出所有组员姓名与分工，并将所有文件打包为单个ZIP提交，便于评分。过程演示视频中也必须包含分工说明，明确个人职责。
2. 应用场景
各小组需选定一个应用领域，围绕其开发连贯的软件系统。所选场景应能体现记录的创建、修改、查询与长期维护等真实业务。
合适示例包括：
学生课程管理系统
图书馆借阅系统
医院患者记录系统
个人财务追踪器
游戏物品/交易系统
健身训练追踪器
宠物管理系统
设备维护系统
（其他同等复杂度领域亦可）
无论选择何种主题，系统都需建模真实世界实体及其关系，反映信息在该场景下的自然组织与使用方式，而非存储无关数据或仅作为孤立函数集合。
3. 初始数据与持久化
程序需表现为真实应用，而非临时脚本。
首次运行：自动初始化工作环境，生成数据集并保存到文件
后续运行：加载并更新已存储数据，不再重新生成
存储数据随使用动态增长与变化
数据集必须满足：
包含大量记录
包含记录间关联关系
程序重启后数据持久化
随用户交互动态更新
禁止使用硬编码小型示例数据集，或每次运行重新生成全新数据。
4. 功能需求
系统需提供与所选领域匹配的连贯操作集合。
用户应能对存储记录进行有意义的交互，而非仅查看静态信息。
系统必须支持：
创建记录
修改记录
删除记录
搜索与查询记录
数据保存与加载
安全处理错误输入
程序应尽可能持续运行，不因普通用户操作错误而崩溃退出。
5. 技术要求
5.1 面向对象设计
程序需按面向对象系统结构设计，为类分配清晰职责。类间关系需反映所建模的真实领域。实现必须体现：
类与对象
封装
继承
多态
组合关系
抽象类/基类
自定义异常
5.2 数据结构
必须自行实现并使用基础数据结构，且需与系统行为深度整合，而非独立实现。
要求实现：
双向链表
栈
队列
树
二叉搜索树
哈希表/映射结构
5.3 算法与递归
系统需体现算法设计思想，而非简单存储与查询。必须包含：
递归树遍历
搜索行为分析
效率与复杂度考量
5.4 文件存储
系统必须将数据持久化存储到文件，重启后恢复上一次状态。
用户可关闭并重启程序，数据不丢失。
5.5 测试
测试应指导开发，而非仅在最后进行。必须包含：
单元测试（独立方法）
类/模块测试
系统测试
测试用于发现缺陷并优化设计。
